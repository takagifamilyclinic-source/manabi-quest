# CLAUDE.md — 個人開発ポートフォリオ棚卸し

## このプロジェクトの目的

増えていく個人開発プロジェクト(アプリ、CLAUDE.mdプロジェクト、指示書、試作)を定期的にスキャンし、「何を持っていて、どれが動いていて、どれが放置されているか」のカタログを維持する。ユーザーが「棚卸しして」と言ったら以下を実行すること。

> このフォルダは**単体で完結するツールキット**です。PC内の任意の場所に置き、Claude Code をこのフォルダで起動して使います。スキャン対象はあなたのPC内の他フォルダ(`config.yaml` で指定)で、このフォルダ自身ではありません。

## 設定

- スキャン対象のルートフォルダは `config.yaml` に記載(複数可)。初回実行時に対話で確認して作成する(`config.example.yaml` を雛形にコピー)
- 除外パターン(node_modules、venv、work/ 等)も config.yaml で管理
- `stale_days`(既定90)で「放置」とみなす日数を調整できる

## 棚卸しの手順

### Step 0:検出スクリプトを走らせる(機械的な事実集め)
- まず `python scan.py` を実行する。これが以下を**決定的に**集めて JSON を返す:
  - ルート直下のプロジェクト候補の列挙(除外適用)
  - CLAUDE.md / README / 指示書系Markdown / ソースの有無
  - 主要言語・ファイル数、最終更新日(git最終コミット or ファイルmtime)
  - 90日以上未更新の ⚫ 予備フラグ、README/セットアップ手順/APIキー疑いの警告
  - 前回スナップショット(`history/` 最新)との差分
- スクリプトは同時に `history/YYYY-MM-DD.json` に生データを保存する(差分比較の材料)
- `config.yaml` が無ければ、ここで対話して作る(下記 Step 1 参照)

### Step 1:プロジェクト検出
- scan.py の出力(`projects[]`)がプロジェクト候補。`is_project=false` は無視してよい
- 判定が怪しいものだけ、ディレクトリを軽く覗いて確認する

### Step 2:各プロジェクトの調査
scan.py が集めた事実を土台に、**判断が要る部分だけ** Claude が補う。README等のドキュメントを優先し、なければコードを軽く読んで推定する(推定した場合はその旨を明記):

- **名前と一言説明**(何をするものか)← README/CLAUDE.md を読んで要約
- **種別**:アプリ / CLAUDE.mdプロジェクト / 指示書のみ(未着手) / 試作・実験(scan.py の `type_hint` を参考に確定)
- **技術**:言語、主要ライブラリ、使用モデル(Sonnet/Fable等)とAPI利用の有無
- **状態**:次の4分類(scan.py は ⚫ しか付けない。🟢🟡🔴 は Claude が判断)
  - 🟢 稼働中(実際に使われている形跡がある)
  - 🟡 動くが未運用
  - 🔴 未完成・中断
  - ⚫ 放置(最終更新から90日以上)
- **最終更新日**(gitがあれば最終コミット、なければファイルmtime)← scan.py の `last_update`
- **ドキュメント健全性**:READMEなし、セットアップ手順なし、APIキーのハードコード等があれば警告 ← scan.py の `doc_health.warnings`

### Step 3:カタログ生成
- `PORTFOLIO.md` を生成・上書きする。構成:
  1. サマリー(総数、状態別内訳、今回の変化)
  2. **前回からの差分**(新規 / 更新あり / 新たに放置入り)— scan.py の `diff` を利用。前回結果は `history/YYYY-MM-DD.json` に保存済み
  3. プロジェクト一覧表(状態順。稼働中→未運用→未完成→放置)
  4. **要判断リスト**:放置プロジェクトごとに「再開 / 完了扱い / アーカイブ」の判断を促す1行

### Step 4:報告
- チャットでサマリーと「要判断リスト」だけを簡潔に報告する。全件の詳細は PORTFOLIO.md に委ねる
- 判断はユーザーに委ねる。「これはもう捨てるべき」等の断定はしない。ただし事実(90日未更新、依存が古い等)は率直に伝える

## アーカイブ運用

- ユーザーが「〇〇をアーカイブ」と言ったら、プロジェクトを `archive/` に移動し、PORTFOLIO.md 上は「アーカイブ済み」セクションに移す(削除はしない)
- アーカイブ時に、そのプロジェクトから得た学び・再利用できる部品を1〜3行でメモに残すよう促す

## してはいけないこと

- ユーザーの確認なしにファイルを移動・削除すること
- コードの中身を必要以上に深追いすること(棚卸しは概要把握が目的。1プロジェクトあたり数ファイルの確認で十分)
- スキャン対象フォルダの外を触ること

## 推奨運用

- 月1回、ガイドライン更新ウォッチと同じタイミングで実行すると習慣化しやすい
- 新しいアイデアに着手する前に一度実行し、「似たものを既に作っていないか」「放置中のものと統合できないか」を確認する使い方も有効
