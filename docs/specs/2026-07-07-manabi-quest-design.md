# まなびクエスト(仮称) 設計書

作成日: 2026-07-07
対象: 小学生の学習支援アプリ(自分のこども向け・低学年1人+中学年1人)

## 目的

- 計算・漢字・英語・理科社会の反復練習をゲーム感覚で続けられるようにする
- つまずいた問題をその場でやさしく解説する
- 毎日の学習習慣をつくる(連続記録・ごほうび)
- 苦手分野を自動で多めに出題し、さらに親+Claude Codeで専用対策パックを作れる

## 絶対条件

- **課金ゼロ**: GitHub Pages無料枠+Pollinations無料経路のみ。従量課金API・有料サービスは一切使わない
- **こどものデータはiPadの外に出さない**: 学習記録・ポイント・図鑑進捗はブラウザのローカル保存のみ。実名は使わない(プレイヤー1/2+ニックネーム)

## 動作方式

- iPad Safariで開き「ホーム画面に追加」→ 全画面PWAとして動作
- Service Workerによりオフラインでも全機能が動く(初回読み込みのみネット必要)
- 配信: GitHub公開リポジトリ+GitHub Pages。更新はgit pushのみで反映
- 本体: `D:\Desktop-Archive\manabi-quest\`(デスクトップ/アプリ配置規約どおり)

## 技術構成

- 素のHTML/CSS/JS(ビルド不要)。PWA(manifest.json+service worker)
- データ保存: localStorage(設定・進捗)+必要に応じIndexedDB(学習ログ)
- 問題データ: JSONファイル同梱。カード画像: PNG同梱(オフライン対応)

## 画面構成(6画面)

1. **プロフィール選択**: 起動時に2人のどちらかを選ぶ(アバター付き)
2. **ホーム**: 今日のミッション・連続日数・レベル・図鑑入口
3. **ドリル**: 教科選択→1セッション10問。大きなタッチボタン。誤答時はその場で「やさしい解き方ガイド」表示
4. **コレクション図鑑**: セッション完了でガチャ券1枚→カード獲得(レア度あり)。図鑑が埋まる
5. **ごほうびポイント帳**: 学習量に応じてポイント加算。現実ごほうびとの交換は親ページで承認
6. **親ページ**: 簡易PINロック。子別履歴・分野別正答率・苦手トップ5・ポイント管理・記録書き出し

## 問題と解説

- **算数**: プログラムで学年別に無限自動生成
  - 低学年: たし算・ひき算(くり上がり/くり下がり)・九九
  - 中学年: わり算・あまりのあるわり算・小数/分数入門・大きな数
  - 解説は問題タイプ別テンプレートで自動付与
- **漢字・英語・理科社会**: 学年別JSON問題パック同梱。1問=問題・選択肢・答え・やさしい解説のセット。配当漢字表など公的基準に準拠して作成
- 全問題に**分野タグ**(例: くり上がりのたし算/3年漢字読み)を付与。苦手分析の単位

## 苦手対応(二段構え)

1. **アプリ内自動調整**: 分野タグごとの正答率を記録(直近成績を重視)し、低正答率の分野の出題率を自動で上げる
2. **Claude Code連携(手動運用)**:
   - 親ページ「記録を書き出し」→ 学習ログJSONをコピー/共有
   - クロード相談用フォルダ等でClaude Codeに渡す
   - Claude Codeが分析し、その子専用の問題パックJSONを作成してリポジトリに追加→git push→iPad側に自動反映(取り込み操作不要)
   - **この手順は完成時にHTML手順書として文書化し、リポジトリに同梱+親ページからリンクする**(忘れ防止・ユーザー要望)

## コレクション=東北モンスター図鑑(郷土×モンスター収集・ユーザー要望)

- 世界観: **ポケモン風のモンスター収集ゲーム**。ただし著作権のため実在キャラは一切使わず、**横手・秋田・東北の名物をモチーフにしたオリジナルモンスター**60〜100体を作る
  - 例: かまくらの雪精霊/なまはげ風の鬼モンスター/竿燈のあかりの妖精/秋田犬のもふもふ獣/いぶりがっこの燻しトカゲ/ハタハタの魚モンスター/鳥海山の岩巨人/ずんだのまめ妖精 など
- **タイプ制**: ゆき/まつり/たべもの/しぜん/でんせつ 等のタイプを付け、図鑑をタイプ別に眺められる
- 各モンスターに**モチーフの豆知識**(例:「かまくらは横手の小正月行事だよ」)を付け、郷土学習を兼ねる
- レア度: ノーマル/レア/スーパーレア/でんせつ の段階
- **進化**(第3弾): 同じ分野の正解を重ねると手持ちモンスターが進化(2段階程度)
- 画像はPollinationsで無料生成し同梱(かわいいゲーム風イラストのトーンで統一)。**生成前にプロンプト一覧を提示して承認を得る**(保管庫ルール)
- **IP注意**: ポケモン等の既存キャラの名称・姿・固有用語(モンスターボール等)は使わない。あくまで「雰囲気が似た文法のオリジナル」に留める

## 見た目・UIの方向性

- 冒険RPG風のゲームUI: ドリル=バトル(正解で敵モンスターにダメージ/セッションクリアで捕獲チャンス)、称号=バッジ風
- 配色・フォントはこども向けゲームらしく明るくポップに。文字は大きく、低学年でも読めるようふりがな配慮
- 効果音・アニメーションはCSS/軽量JSの範囲で(外部ライブラリ・素材課金なし)

## やる気の仕組み

- **コレクション(核)**: セッション完了→ガチャ演出→モンスター捕獲
- **連続記録**: 毎日実施で連続日数が伸びる。カレンダースタンプ
- **レベル・称号**: 経験値でレベルアップ、節目で称号
- **現実ごほうび連動**: ポイント帳。交換は親ページで承認・消込

## データ設計(概要)

- `profiles`: [{id, nickname, grade, avatar}]
- `attempts`: [{profileId, skillTag, questionId?, correct, timestamp}]
- `progress`: {profileId: {xp, level, streak, lastPlayedDate, points, cards[]}}
- 問題パック: `packs/<subject>-<grade>.json` = [{id, skillTag, q, choices[], answer, explanation}]
- 専用パック: `packs/custom-<profileId>-<date>.json`(Claude Code生成)

## エラー・例外への備え

- オフライン時: 全機能動作(SWキャッシュ)。更新は次回オンライン時
- ローカル保存の消失リスク: 親ページの「記録書き出し」が実質バックアップを兼ねる。書き出しを定期的に促す表示
- iPad Safari固有の挙動(ホーム画面追加時のストレージ分離等)は実機確認する

## テスト

- PC上でPlaywright/Chrome DevToolsによる画面・ロジック確認
- 出題生成ロジックはNode標準のテストスクリプトで検証(学年範囲・答えの正しさ)
- 最終的にiPad実機で: オフライン動作・データ永続・タッチ操作を確認

## 作る順番(3段階)

1. **第1弾(MVP)**: プロフィール切替+算数自動生成ドリル(バトル風)+セッション→捕獲→図鑑(初期モンスター20体程度)+連続日数 → こどもに触ってもらい反応を見る
2. **第2弾**: 漢字パック+親ページ(PIN・苦手分析)+苦手自動調整
3. **第3弾**: 英語・理科社会パック+レベル称号(バッジ)+進化+ごほうびポイント帳+記録書き出し+**Claude連携HTML手順書**

## スコープ外(今回はやらない)

- 手書き文字認識(漢字の書き取り判定は選択式・なぞり表示で代替)
- 音声読み上げ(将来検討: Web Speech APIなら無料で追加可能)
- 複数iPad間の同期
